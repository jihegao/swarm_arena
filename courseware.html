🎯 游戏目标

⏱ 时间限制
游戏在 5000 tick 后结束（或只剩一种生物时提前结束）。
🏆 胜利条件
存活数量最多的物种中，能量最高的个体获胜。
🦁 核心玩法
抢食物 → 长大 → 吃掉比自己小的对手 → 繁殖后代 → 数量碾压。
📊 计分规则
吃掉对手 +对手max_energy | 吃食物 +amount | 繁殖 +100 | 最终获胜 +500

关键思路：这是一个滚雪球游戏。前期抢到食物 → 体型变大 → 能吃更多对手 → 获得更多能量 → 繁殖更多后代。起步很关键！

这段代码创建了一个只会直线前进的橙色生物。先运行观察，再把一条明确假设改进进去。

👁 Perception — 你能看到什么

每一帧，游戏引擎调用 decide(perception)，传入感知信息：

字段类型说明
self_x / self_yfloat你的坐标
self_anglefloat你的朝向（弧度，0=右，pi/2=下）
self_sizefloat你的体型大小（随能量动态变化）
self_energyfloat你的能量值
self_max_energyfloat你的初始能量值
world_width / heightint世界尺寸
nearby_foodlist[FoodSpot]视野范围内的食物列表
nearby_creatureslist[CreatureSpot]视野范围内的其他生物
FoodSpot
x, y 坐标 | amount 能量值 | size 大小
CreatureSpot
x, y 坐标 | creature_type 类型名 | size 体型 | energy_pct 能量百分比

你只能看到 vision_radius 范围内的东西，范围外的世界不可见！

🎯 Action — 你要做什么

每帧返回一个 Action 对象：

Action(
    target_angle,       # float: 这一帧要朝向的角度（弧度）
    target_speed,       # float: 这一帧的移动速度（不超过 max_speed）
    attack_target_id,   # int | None: 要追击的生物 ID（仅标记意图）
    reproduce,          # bool: 是否繁殖（默认 False）
)
🏃 移动
每帧必须指定 target_angle 和 target_speed。设 0 表示停止。
⚔️ 攻击
碰到比自己小的生物会自动吃掉，无需主动声明攻击！
👶 繁殖
设 reproduce=True 且满足条件时，附近生成同类型子代。

战斗规则很简单：大吃小。没有攻击力和防御力。你只需要移动到比你小的生物旁边就行！

⚙️ 属性系统与点数分配

可配置属性

属性默认最小最大说明
max_speed1.00.54.0最大移动速度
max_energy10030200初始能量值
vision_radius2050300视野半径

点数预算 = 1.5

# 归一化值 = (值 - 最小) / (最大 - 最小)
# 总点数 = 各属性归一化值之和
# 超过 1.5 则等比例缩小

# 示例：Hunter 的分配
speed  = (2.5 - 0.5) / (4.0 - 0.5) = 0.57
energy = (80  - 30) / (200 - 30) = 0.29
vision = (200 - 50) / (300 - 50) = 0.60
总计   = 1.46

速度最关键：能吃到食物就能长大，长大就能吃别人，是滚雪球的核心！

🍖 大吃小规则

战斗规则

  • 碰到比自己体型小的生物，直接吃掉
  • 获得对方全部剩余能量
  • 碰撞范围：距离 < (eater.size + target.size) × 0.6
  • 吃掉后 10 帧冷却

体型公式

size = 6 × sqrt(energy / 100)

👶 繁殖机制

繁殖条件

条件值
能量阈值≥ max_energy × 2.0
繁殖消耗max_energy × 0.5
冷却时间120 帧
子代位置母体周围随机 20px
世界上限600 生物

流程

  1. reproduce=True
  2. 检查能量 ≥ max_energy × 2
  3. 检查冷却为 0
  4. 扣除能量，生成子代
  5. 进入 120 帧冷却，得 100 分

繁殖策略

⏰ 早生策略
能量刚到阈值就生，子代小但数量多。
💪 晚生策略
攒更多能量再生，子代更大更安全。
🔭 时机选择
周围没有天敌时繁殖更安全。

判断是否可繁殖：perception.self_energy >= perception.self_max_energy * 2.0

⚡ 能量系统

能量是生存的核心。能量降到 0 你就死了。

能量消耗（每帧）

来源公式
基础衰减0.1 × (体型 / 6)
移动消耗额外 0.08 × (速度/max_speed) × (体型/6)

体型越大，能量消耗越高！没有食物来源时，适当"减肥"可能比维持巨大体型更持久。

能量恢复

来源效果
🍞 吃食物+ 食物的 amount（8~100）
🦁 吃掉对手+ 对手全部剩余能量
👶 繁殖- max_energy × 0.5

吃食物条件

你与食物的距离 < 你的体型 + 3

🔵 PackHunter（群猎者）
速度 2.2 | 能量 80 | 视野 150
寻找同伴，聚在一起时围猎。附近同类 <5 且能量充足时繁殖。落单时逃跑。

🟡 Scavenger（拾荒者）
速度 3.5 | 能量 60 | 视野 160
全场最快，抢到食物就跑。能量超 3 倍时繁殖。从不主动攻击。
弱点：初始能量最低，谁都能吃它。

🧬 GA 参数优化怎么讲

GA 像一次“参数选拔赛”：规则已经有了，我们只是让很多组数字上场比较，留下更适合目标的那一组。

1. 多组参数速度、视野、逃跑距离、繁殖阈值各不相同
→
2. 模拟比赛每组参数都放进同一个世界里跑一遍
→
3. 看表现按生存时间、能量、数量或得分比较
→
4. 新一代留下高分参数，组合并做一点随机微调
学生要看懂什么
不是“机器变聪明了”，而是同一套行为规则换了更合适的数字。训练后要比较旧参数和新参数。
max_speed vision_radius flee_distance reproduce_ratio
课堂提问
“这组参数为什么可能解决刚才的观察？” 例如速度更快是否真的抢到更多食物，繁殖更晚是否真的让后代更安全。

🧠 Q-learning 怎么讲

Q-learning 像一本“动作经验表”：生物看到当前状态，试一个动作，拿到反馈，然后把这次经验记下来。

1. 看状态附近有没有食物、威胁、猎物，能量是否充足
→
2. 选动作找食物、逃跑、追击、漫步、繁殖
→
3. 得反馈能量变多、活得更久是好反馈；死亡是坏反馈
→
4. 改表格下次遇到类似状态，更倾向高分动作

📝 完整示例 A：规则 Creature

规则路线适合“看到 X 就做 Y”。这个例子直接用 decide() 写出逃跑、追击、找食物、繁殖。

class RuleSurvivor(Creature):
    def __init__(self, x, y):
        super().__init__(x, y, "RuleSurvivor", (255, 128, 0))
        self.max_speed = 2.2
        self.vision_radius = 160

    def decide(self, p):
        if p.self_energy >= p.self_max_energy * 2.5:
            return Action(p.self_angle, 0, reproduce=True)
        bigger = [c for c in p.nearby_creatures if c.size > p.self_size]
        if bigger:
            enemy = min(bigger, key=lambda c: math.hypot(c.x-p.self_x, c.y-p.self_y))
            return Action(math.atan2(p.self_y-enemy.y, p.self_x-enemy.x), self.max_speed)
        # 然后追小生物；没目标时找食物或漫步。

教学重点：先让学生说出规则，再把一条规则翻译成代码。

🧠 完整示例 C：RL Creature

RL 路线先训练 best_policy.json，再让生物读取 Q 表，根据当前状态选最高分动作。

from training.policy import decode_action, discrete_state_key, encode_features
from training.rl import load_q_policy

class PolicyTableSurvivor(Creature):
    _policy = None
    def __init__(self, x, y):
        super().__init__(x, y, "PolicyTableSurvivor", (190, 120, 255))
        self.max_speed, self.vision_radius = 2.2, 180
        if PolicyTableSurvivor._policy is None:
            root = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
            PolicyTableSurvivor._policy = load_q_policy(os.path.join(root, "best_policy.json"))
    def decide(self, p):
        state = discrete_state_key(encode_features(p))
        action = PolicyTableSurvivor._policy.select_action(state, explore=False)
        return decode_action(action, p, self)

运行：python3 train.py --method rl --episodes 200 --output best_policy.json
